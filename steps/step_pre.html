2. GenomeStudio vs DRAGEN Array — Tool Selection

Two tools can convert Illumina IDAT files into genotype calls. This project used GenomeStudio. DRAGEN Array (versions 4.4.4 and 4.5.4, confirmed on Biotech2024) was available but not used for this dataset.

CriterionGenomeStudio (used here)DRAGEN Array (available, not used)
Function Full GUI: loads IDATsvia sample sheet, applies .egt cluster file, GenCall algorithm for AA/AB/BB assignment, BAF/LRR visualisation, manual operator review, PLINK/VCF export Command-line pipeline: IDAT → VCF/GTC/PLINK via same GenCall algorithm. Batch processing on DRAGEN hardware accelerator. No interactive review.
Operator review✓ Per-sample QC, BAF/LRR tracks, manual chip failure flagging✗ Fully automated, no visualisation
d/t replicates✓ Operator assigns d/t suffixes in project before export✗ No concept of d/t; must be post-processed
Failed chip detection✓ Visual per-chip call rate and BAF inspectionPer-sample call rate metrics only; no chip-level view
Throughput✗ GUI-based, Windows only, slow for large batches✓ Linux-native, hardware-accelerated, scriptable
ReproducibilityOperator decisions not auto-logged; must recover from .gtProject✓ Fully scriptable; complete command-line audit trail
When to useFirst-pass analysis requiring operator review; contamination/failed chip/d/t detectionRe-analysis with fixed parameters; high-throughput production; when audit trail required
Why GenomeStudio was used. The ALSU dataset required operator decisions before export: 48 duplicate-ID samples needed d/t suffix assignment; 4 failed chips needed visual confirmation via BAF/LRR; 3 contaminated samples needed flagging. All require interactive review. For a future re-analysis with fixed parameters, DRAGEN Array would be preferable for reproducibility and speed.

4. GenomeStudio Operations (Reconstructed)

4.1 Sample sheet and d/t assignment

The sample sheet (1,248 rows, 1,200 unique Sample_IDs) maps chip barcodes and row/column positions to sample IDs. 48 rows have duplicate IDs (replicate experiments). The operator assigned d and t suffixes to disambiguate these in GenomeStudio before export.

4.2 Cluster file and GenCall

GenomeStudio applies a cluster file (.egt) defining expected signal distributions for AA/AB/BB clusters at each probe. Each call is scored 0–1 (GenCall score); calls below the threshold (default 0.15, value used here unknown) are coded as missing. The cluster file is not on Biotech2024 and must be recovered — it determines F_MISS values directly.

6. Open Items

CRITICAL — Cluster file (.egt). Must be recovered from the Windows machine (D:\perenos\ALSU-analysis\). Without it, genotype calling cannot be reproduced. It defines all cluster positions and directly controls F_MISS values.
CRITICAL — GenomeStudio project (.gtProject). Contains the cluster file reference, GenCall threshold, sample sheet path, and all operator decisions. Essential for methods section and audit trail.
IMPORTANT — GenCall score threshold. Default is 0.15 but any non-default value must be documented. Found in GenomeStudio project settings. This threshold directly determines which calls are coded as missing (F_MISS values).
IMPORTANT — Strand convention. TOP/BOT vs Forward/Reverse. Affects allele coding in downstream association tests and imputation. Verify by comparing a known probe in ConvSK_raw.bim against the .bpm manifest.
IMPORTANT — BAF verification for 3 contaminated samples (08-25, 08-495, 08-701). Contaminated samples show 5-band BAF pattern (≈ 0, 0.25, 0.50, 0.75, 1.0) vs normal 3-band. FullDataTable.txt on Biotech2024 contains BAF values and can be used without the GUI.
RECOMMENDED — GenomeStudio and Genotyping Module versions. Recoverable from .gtProject file header. Required for publication methods section.
