1 Overview

Raw Illumina array data passes through two processing stages before entering the PLINK pipeline. The first converts scanner output into per-sample genotype calls; the second assembles those calls, performs operator-level QC, and exports a PLINK dataset. This page documents both stages, what is confirmed from files on Biotech2024, and what remains to be recovered.

Output of this step: ConvSK_raw — 654,027 variants × 1,247 samples, GRCh37 (hg19), Illumina GSA-24v3-0_A2. Verified by direct file count on Biotech2024: wc -l ConvSK_raw.fam → 1247 and wc -l ConvSK_raw.bim → 654027.

1. Illumina iScan — raw intensity scanning
Hybridised BeadChips scanned at two wavelengths. Produces Red.idat + Grn.idat per sample position. 56 chip directories confirmed on Biotech2024 (including 4 failed chips).
Location: /staging/ALSU-analysis/Conversion/OUTPUT/<barcode>/ · <barcode>_R##C##_Grn.idat / _Red.idat
2. GenomeStudio — genotype calling & QC
Loads all IDAT files + sample sheet. Applies cluster file (.egt) to assign AA/AB/BB genotypes via GenCall algorithm. Operator reviews call rates, BAF/LRR tracks, assigns d/t suffixes to replicate IDs. Exports PLINK files. Run on Windows (project path: D:\perenos\ALSU-analysis\...), not on Biotech2024.
Output: ConvSK.ped + ConvSK.map → converted to ConvSK_raw.bed/.bim/.fam
3. ConvSK_raw — pipeline entry point
PLINK binary dataset. Verified on Biotech2024. Immutable — all downstream steps use symlinks.
/staging/ALSU-analysis/winter2025/PLINK_301125_0312/ConvSK_raw.{bed,bim,fam}

2 GenomeStudio vs DRAGEN Array — Tool Selection

Two tools can convert Illumina IDAT files into genotype calls. This project used GenomeStudio. DRAGEN Array is also installed on Biotech2024 (versions 4.4.4 and 4.5.4 confirmed) and can perform the same conversion, but was not used for this dataset. The table below explains both tools and the criteria for choosing between them.

Criterion GenomeStudio (used here) DRAGEN Array (available, not used)
What it does Full GUI workflow: loads IDATsvia sample sheet, clusters probes using a .egt cluster file, assigns genotype calls (GenCall score), displays BAF/LRR tracks, allows manual review and operator intervention, exports to PLINK/VCF/CSV Command-line pipeline: converts IDAT → VCF (or GTC/PLINK) via the same GenCall algorithm, optimised for batch processing. No graphical review. Runs on the DRAGEN accelerator hardware.
Operator review ✓ Manual per-sample and per-probe QC; BAF/LRR visualisation; d/t suffix assignment by operator ✗ Fully automated; no interactive review
Cluster file Uses supplied .egt cluster file. Can recluster on the loaded sample set for improved accuracy. Uses the same .egt cluster file. No interactive reclustering.
d/t replicate handling ✓ Operator assigns d/t suffixes directly in GenomeStudio during project setup ✗ No concept of d/t; must be handled in post-processing
Failed chip identification ✓ Visual inspection of call rates and BAF plots per chip; operator can flag chips Automated: per-sample call rate metrics in output VCF/report; no chip-level visualisation
Throughput ✗ GUI-based; slower for large batches; Windows only ✓ Command-line batch; DRAGEN hardware acceleration; Linux-native on Biotech2024
Reproducibility Operator decisions (cluster review, manual exclusions) are not automatically logged; must inspect .gtProject file ✓ Fully scriptable; all parameters in command line; complete audit trail in logs
Output format PLINK PED/MAP (used here), VCF, CSV final report VCF (primary), GTC, PLINK (via post-processing)
When to use First-pass analysis where operator review is important; clinical cohorts; datasets with known QC issues (contamination, d/t replicates, failed chips) requiring visual inspection High-throughput production genotyping; re-analysis of large cohorts where parameters are fixed; when full reproducibility logging is required
ℹ
Why GenomeStudio was used for this project. The ALSU dataset required operator-level decisions: 48 duplicate-ID samples needed d/t suffix assignment to distinguish replicate experiments before export; potential failed chips needed visual confirmation via BAF/LRR tracks; contamination suspects needed to be flagged. These decisions require interactive review, which GenomeStudio provides and DRAGEN Array does not. For a future re-analysis with fixed parameters, DRAGEN Array would be the better choice for reproducibility and speed.

4 GenomeStudio Operations (Reconstructed)

GenomeStudio was run on a Windows machine. The following describes the operations performed, reconstructed from the output files and the PLINK export command in ConvSK.bat on Biotech2024.

4.1 Sample sheet loading

The sample sheet (1248_merged_sample_sheet_12022025.csv.csv, 1,248 rows) maps chip barcodes and row/column positions to sample IDs. 1,200 unique Sample_IDs are present; 48 rows have duplicate IDs corresponding to d/t replicate experiments on different physical chips. The operator assigned d and t suffixes to disambiguate these within GenomeStudio before export.

4.2 Cluster file and GenCall

GenomeStudio applies a cluster file (.egt) to define the expected signal distributions for the three genotype clusters (AA, AB, BB) at each probe. Each call is assigned a GenCall score (0–1); calls below the GenCall threshold (default 0.15, value used here unknown) are coded as missing. The specific cluster file used is not on Biotech2024 and must be recovered — it determines call rates and F_MISS values directly.

4.3 PLINK export

The PLINK Report Plug-in exported the dataset as PED/MAP format (ConvSK.ped, ConvSK.map), subsequently converted to binary BED format. Export settings confirmed from files:

  • Genome build: GRCh37 / hg19 (confirmed from .bim chromosomal coordinates)
  • Sample count: 1,247 (1,200 unique IDs + 47 d/t entries — one triplet counted differently)
  • Variant count: 654,027 (confirmed from .bim)
  • Strand convention: unknown — must be confirmed from manifest comparison
# ConvSK.bat — the original export command (recovered from Biotech2024):
"C:\plink\plink-1.04-dos\plink.exe" --script D:\perenos\ALSU-analysis\Conversion\OUTPUT\ConvSK\PLINK_301125_0312\ConvSK.script

6 Open Items

CRITICAL

Recover the cluster file (.egt) from the Windows machine where GenomeStudio ran. Without this, genotype calling cannot be reproduced. The .egt file defines all cluster positions and directly determines F_MISS values. Check the Windows project directory at D:\perenos\ALSU-analysis\.

CRITICAL

Recover the GenomeStudio project file (.gtProject) from the Windows machine. It records the cluster file used, GenCall threshold, sample sheet, and all operator decisions (manual exclusions, d/t suffix assignments). Essential for a complete audit trail.

IMPORTANT

Confirm the GenCall score threshold. Default is 0.15 but any non-default value must be documented. Found in GenomeStudio project settings or operator notes. This threshold directly controls what is coded as missing and therefore determines F_MISS values.

IMPORTANT

Confirm the strand convention (TOP/BOT vs Forward/Reverse) used in the PLINK export. This affects allele coding in downstream association tests and imputation. Verify by comparing a known probe's alleles in ConvSK_raw.bim against the GSA-24v3-0_A2.bpm manifest.

IMPORTANT

Verify BAF plots for 3 contaminated samples (08-25, 08-495, 08-701) in GenomeStudio. A contaminated sample shows a 5-band BAF pattern (≈ 0, 0.25, 0.50, 0.75, 1.0) instead of the normal 3-band pattern. The FullDataTable.txt on Biotech2024 contains BAF values and can be used for this check without the GUI.

RECOMMENDED

Confirm GenomeStudio and Genotyping Module versions from the .gtProject file header. Required for the methods section of any publication.

RECOMMENDED

Confirm scan date and scanner model from IDAT file metadata. IDAT files store scan parameters in a binary header; FullDataTable.txt may also contain this information in the header rows.
