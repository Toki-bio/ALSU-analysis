Step 1: Sample Missingness Filter

Remove samples with catastrophic genotyping failure before any SNP-level statistics are computed

★ Completed — Spring 2026 (re-analysis)

1. Overview

Per-sample missingness (F_MISS) is the fraction of genotype calls that failed for a given individual. Elevated missingness indicates technical failures: degraded DNA, low input concentration, partial array hybridisation, or scanning artefacts. Retaining such samples introduces differential missingness bias that produces spurious associations and inflated test statistics in all downstream analyses.

Rationale: Sample-level filtering comes before SNP-level QC so that high-missingness samples do not bias HWE p-values, allele frequency estimates, or per-SNP call rates computed later. A relaxed 20% threshold removes only catastrophic failures while retaining as much of the Central Asian cohort as possible.

★ Spring 2026 re-analysis. This page documents the corrected run (99 removed → 1,148 retained). The original winter 2025 pipeline removed only 92 samples — 7 with F_MISS > 0.20 were silently missed and propagated through Steps 2–15. The root cause was never identified. The 7 extra samples are listed in Section 5.

2. QC Parameter: Per-Sample Missingness

PLINK computes F_MISS = N_MISS / N_GENO. The --mind flag removes individuals above the threshold.

Note on N_GENO. PLINK internally excludes multiallelic sites, duplicate positions, and I/D alleles from the missingness calculation. The effective N_GENO in ConvSK_raw_miss.imiss is 620,492, not 654,027. All F_MISS values on this page use this denominator.
ParameterValue usedTypical rangeInterpretation
--mind 0.2020%0.01–0.10 Removes samples failing >124,000 of 620,492 calls. Very relaxed — only catastrophic failures removed.
ThresholdStringencyUse case
--mind 0.01Very strictLarge GWAS, high genotyping quality (e.g. UK Biobank)
--mind 0.02StrictAnderson et al. 2010 (Nature Protocols) standard
--mind 0.05ModeratePopulation genetics studies
--mind 0.10RelaxedClinical cohorts with variable DNA quality
--mind 0.20Very relaxedUsed here — removes only catastrophic failures, maximises cohort size

7. Commands Executed

All commands run on Biotech2024 (Oracle Linux 8.6), user copilot, path /staging/ALSU-analysis/spring2026/.

Step 1a: Set up workspace

mkdir -p /staging/ALSU-analysis/spring2026/
cd /staging/ALSU-analysis/spring2026/
ln -s ../winter2025/PLINK_301125_0312/ConvSK_raw.bed ConvSK_raw.bed
ln -s ../winter2025/PLINK_301125_0312/ConvSK_raw.bim ConvSK_raw.bim
ln -s ../winter2025/PLINK_301125_0312/ConvSK_raw.fam ConvSK_raw.fam
wc -l ConvSK_raw.fam   # 1247
wc -l ConvSK_raw.bim   # 654027

Step 1b: Compute per-sample missingness

plink --bfile ConvSK_raw       --missing       --out ConvSK_raw_miss
# Produces ConvSK_raw_miss.imiss and .lmiss
# N_GENO in .imiss = 620,492 (PLINK excludes multiallelic/duplicate/I/D probes)

Step 1c: Extract removal list (F_MISS > 0.20)

awk 'NR>1 && $6+0 > 0.20 {print $1"	"$2}'     ConvSK_raw_miss.imiss > remove_miss20.txt
wc -l remove_miss20.txt   # 99 (verified on Biotech2024)

Step 1d: Remove high-missingness samples

plink --bfile ConvSK_raw       --remove remove_miss20.txt       --make-bed       --out ConvSK_mind20
# Log: 99 people removed, 1148 remaining, 654027 variants and 1148 people pass filters.
