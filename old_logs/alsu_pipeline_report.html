Pipeline Overview

STAGE 1: Quality Control & Sample Filtering

Raw PLINK Dataset

  • Samples: 1,247
  • Variants: 654,027
  • Platform: Illumina Array

Filter Applied: Sample Missingness (F_MISS > 0.20)

Result: 1,155 samples (−92 high-missingness samples)

↓
STAGE 2: Identity-by-Descent Deduplication

LD Pruning for IBD

  • Input: 145,120 variants (MAF≥0.1, autosomal, high-quality)
  • LD pruning: window 50, step 5, r² < 0.1
  • Output: 44,782 independent variants

IBD Computation (PLINK --genome)

  • PI_HAT ≥ 0.98 threshold
  • Duplicate pairs detected: 65
  • Duplicate clusters: 49
  • Strategy: Keep 1 per cluster (lowest F_MISS)

Result: 1,098 unique samples (−57 duplicates removed)

↓
STAGE 3: VCF Preparation & Encoding Fixes

SNP Quality Control

  • Removed I/D allele codes: 494 variants
    → PLINK shorthand (I=Insertion, D=Deletion) invalid in VCF format; requires A/C/G/T nucleotides
  • VCF-safe variants: 472,191

VCF Processing

  • Concatenated chromosomes 1-22
  • Filtered to biallelic SNPs only
  • Removed 396 duplicate positions
    → Multiple variants at same CHR:POS from array design artifacts; Michigan requires unique positions
Critical Fix: Cyrillic Homoglyph Correction
  • 03-25м → 03-25m (Cyrillic 'м' to ASCII 'm')
  • 08-176Х-00006 → 08-176X-00006 (Cyrillic 'Х' to ASCII 'X')

Final Pre-Imputation VCF: 1,098 samples, 472,191 variants

↓
STAGE 4: Michigan Imputation Server

Configuration

  • Reference: 1000 Genomes Phase 3 v5 (GRCh38)
  • Population: ALL (multi-ancestry)
  • Phasing: Eagle v2.4

QC Results

  • Samples passed: 1,098 ✓
  • Strand flips: 0 ✓
  • Allele switches: 0 ✓
  • A/T C/G palindromic SNPs: 0 ✓

Output: 58,886,952 total imputed variants

STAGE 5: Post-Imputation Processing

Dosage-Based Allele Frequency Derivation

  • Method: AF = Σ(DS) / (2 × N_non-missing)
  • Per-chromosome UZB_chr*.AF_DS.tsv files
  • Joined with R² from chr*.info.gz

High-Quality Variant Selection

  • Criteria: IMPUTED==1 & R²≥0.8 & MAF≥0.001
Final Result: 10,009,530 high-quality variants with Uzbek allele frequencies

Conclusions

This pipeline successfully processed 1,247 Uzbek individuals through comprehensive quality control, imputation, and post-imputation filtering:

  1. Rigorous pre-imputation QC (92 samples removed for missingness, 57 duplicates removed)
  2. Careful VCF preparation (homoglyph fix, strand alignment, I/D removal)
  3. High-quality imputation via Michigan Server (58.9M variants, 1000G Phase 3 reference)
  4. Post-imputation QC (24 heterozygosity outliers removed)
  5. Population-specific allele frequency derivation for Uzbek cohort
Final Achievement: The resulting dataset of 1,074 high-quality samples with 10,846,569 variants represents a foundational resource for:
  • Pregnancy loss genetic association studies
  • Central Asian population genetics
  • Rare variant discovery in an understudied population
  • Population structure analyses (PCA, ADMIXTURE)
  • Runs of homozygosity and inbreeding studies
  • Comparative genomics with reference populations
